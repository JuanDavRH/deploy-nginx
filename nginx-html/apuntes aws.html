  • La readquisición implica reemplazar una aplicación existente por una versión basada en la nube, como el software que se encuentra en AWS Marketplace.

  • El realojamiento implica trasladar una aplicación a la nube y realizar pocas modificaciones a la propia aplicación o no aplicarle ningún cambio. También se lo conoce como “lift and shift”.

  • El cambio de plataforma implica la optimización selectiva de los aspectos de una aplicación para obtener beneficios en la nube sin cambiar la arquitectura central de la aplicación. También se conoce como “lift, tinker, and shift”.

    • El pilar de excelencia operativa incluye la capacidad de ejecutar cargas de trabajo de manera eficaz, obtener información sobre las operaciones y mejorar continuamente los procesos de respaldo para ofrecer valor de negocio.

    • El pilar de seguridad se centra en la protección de datos, sistemas y activos. También se centra en el uso de las tecnologías en la nube para mejorar la seguridad de sus cargas de trabajo.

    • El pilar de fiabilidad se centra en la capacidad de una carga de trabajo de realizar de manera uniforme y correcta las funciones previstas.

  • AWS Outposts es un servicio que le permite ejecutar infraestructura en un enfoque de nube híbrida.

  • Amazon CloudFront es un servicio de entrega de contenido. Utiliza una red de ubicaciones de borde para almacenar contenido en caché y entregarlo a los clientes de todo el mundo. Cuando el contenido se almacena en caché, lo hace localmente como una copia. Este contenido podría ser archivos de video, fotos, páginas web, entre otros.

  • AWS Snowball es un dispositivo que le permite transferir grandes cantidades de datos dentro y fuera de AWS.

  • Un balanceador de carga actúa como un único punto de contacto para todo el tráfico web que entra en el grupo de Auto Scaling. Es decir que, a medida que se agregan o eliminan las instancias de Amazon EC2 en respuesta a la cantidad de tráfico entrante, estas solicitudes se dirigen primero al balanceador de carga y luego se distribuyen entre varios recursos que las gestionarán.

  • AWS Direct Connect es un servicio que le permite establecer una conexión privada dedicada entre su centro de datos y una VPC.

  • Las instancias reservadas son un descuento de facturación que se aplica al uso de instancias bajo demanda en su cuenta. Puede comprar instancias reservadas estándar y convertibles por un plazo de 1 o 3 años, e instancias reservadas programadas por un plazo de 1 año.

  • Amazon EKS es un servicio completamente administrado que puede utilizar para ejecutar Kubernetes en AWS. Kubernetes es un software de código abierto que le permite implementar y administrar aplicaciones en contenedores a gran escala.
  • Amazon EBS ofrece volúmenes de almacenamiento a nivel de bloques que puede utilizar con las instancias de Amazon EC2. Si detiene o termina una instancia de Amazon EC2, todos los datos del volumen de EBS asociado permanecen disponibles.
  • Amazon Route 53 es un servicio web de DNS. Proporciona a los desarrolladores y los negocios una manera fiable de dirigir a los usuarios finales a las aplicaciones de Internet que se alojan en AWS. Además, puede transferir registros de DNS para los nombres de dominio existentes cuya administración actualmente está a cargo de otros registradores de dominio o bien registrar nuevos nombres de dominio de forma directa en Amazon Route 53.
  • Amazon SQS es un servicio de cola de mensajes. Con Amazon SQS, puede enviar, almacenar y recibir mensajes entre componentes de software de cualquier volumen, sin pérdida de mensajes ni la necesidad de que otros servicios estén disponibles.
  • Con la interfaz de línea de comandos de AWS (CLI de AWS), puede controlar varios servicios de AWS directamente desde la línea de comandos dentro de una herramienta. Por ejemplo, puede utilizar comandos para lanzar una instancia de Amazon EC2, conectar una instancia de Amazon EC2 a un grupo de Auto Scaling específico, etc. La CLI de AWS está disponible para los usuarios de Windows, macOS y Linux.
  • Un grupo de seguridad es un firewall virtual que controla el tráfico entrante y saliente de una instancia de Amazon EC2. De forma predeterminada, un grupo de seguridad deniega todo el tráfico entrante y permite todo el tráfico saliente. Puede agregar reglas personalizadas para configurar qué tráfico debe permitirse o denegarse.

  • Amazon CloudWatch es un servicio que proporciona datos que puede utilizar para monitorear sus aplicaciones, optimizar la utilización de recursos y responder a los cambios de rendimiento en todo el sistema.

  • Por medio de CloudTrail, puede ver un historial completo de la actividad del usuario y las llamadas a la API de sus aplicaciones y recursos. Por lo general, los eventos se actualizan en CloudTrail dentro de los 15 minutos posteriores a la realización de una llamada a la API. Puede filtrar los eventos y especificar la hora y la fecha en que se produjo una llamada a la API, el usuario que solicitó la acción, el tipo de recurso que participó en la llamada a la API, etc.

Eso contribuye a ofrecer interconectividad para respaldar los servicios y las aplicaciones que se ejecutan dentro de una región.

Si accede a un objeto en el nivel de acceso poco frecuente, Amazon S3 lo traslada de forma automática al nivel de acceso frecuente, S3 Estándar.
